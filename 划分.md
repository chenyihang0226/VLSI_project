## 1. OpenMP 并行化改造

### 1.1 改造动机

原实现使用 `std::async` + `std::future` + `std::mutex` 实现多线程并行，代码约 40 行，包括手动分批调度、future 同步、互斥锁保护等。改用 OpenMP 后代码缩减为约 10 行，且获得自动负载均衡。

### 1.2 头文件变更

**删除**（不再需要）：
```cpp
#include <future>     // std::async, std::future
#include <mutex>      // std::mutex, std::lock_guard
#include <thread>     // std::thread::hardware_concurrency
```

**新增**：
```cpp
#include <omp.h>      // OpenMP: #pragma omp parallel for, omp_get_max_threads()
```

### 1.3 并行核心代码对比

**改造前（std::async）**：
```cpp
mutex best_mutex;
vector<char> best_part;
int best_cut = numeric_limits<int>::max();

unsigned int concurrent = min(num_threads, (unsigned int)kRestarts);

for (int batch_start = 0; batch_start < kRestarts; batch_start += concurrent) {
    int batch_end = min(batch_start + (int)concurrent, kRestarts);
    vector<future<RestartResult>> futures;
    for (int r = batch_start; r < batch_end; ++r) {
        futures.push_back(async(launch::async, [r, &fine, deadline]() {
            return run_one_restart(r, fine, deadline);
        }));
    }
    for (auto &f : futures) {
        RestartResult result = f.get();
        lock_guard<mutex> lock(best_mutex);
        if (result.cut < best_cut) {
            best_cut = result.cut;
            best_part = move(result.part);
        }
    }
}
```

**改造后（OpenMP）**：
```cpp
vector<RestartResult> results(kRestarts);

#pragma omp parallel for num_threads(num_threads) schedule(dynamic)
for (int r = 0; r < kRestarts; ++r) {
    results[r] = run_one_restart_kway(r, fine, k, deadline);
}

vector<int> best_part;
int best_cut = numeric_limits<int>::max();
for (auto &res : results) {
    if (res.cut < best_cut) {
        best_cut = res.cut;
        best_part = move(res.part);
    }
}
```

### 1.4 关键变化

| 项目 | std::async | OpenMP |
|------|-----------|--------|
| 并发控制 | 手动分批 | `num_threads()` 子句 |
| 负载均衡 | 静态分批（尾部等待） | `schedule(dynamic)` 自动均衡 |
| 结果同步 | `std::mutex` 互斥锁 | **无锁**（预分配数组 `results[r]`，`r` 私有） |
| 线程管理 | `launch::async` 创建 | 编译器生成线程代码 |
| 代码量 | ~40 行 | ~10 行 |

### 1.5 变量共享分析（OpenMP 规则）

| 变量 | 属性 | 原因 |
|------|------|------|
| `r`（循环变量） | **private** | OpenMP 自动私有化 |
| `results[]` | **shared** | 各线程写不同下标 `results[r]`，天然无竞争 |
| `fine` | **shared** | 只读，安全 |
| `deadline` | **shared** | 只读（time_point 是 64-bit 整数） |
| `run_one_restart_kway` 内部 | **thread-local** | 函数局部变量在栈上分配 |

## 2. 自适应时间预算

### 2.1 改造动机

固定 90 秒 deadline 对小图（ibm01, 12k 节点）足够，但对大图（case1, 300k 节点）远远不够，导致所有线程在粗化阶段就被迫退出。

### 2.2 常量定义

**删除**：
```cpp
constexpr int kTotalTimeBudgetSeconds = 90;
```

**新增**：
```cpp
constexpr int kBudgetBaseTime = 90;        // 基准时间（秒），对应 ibm01 ~12k 节点
constexpr int kBudgetBaseNodes = 12000;    // 基准节点数
constexpr double kBudgetExponent = 0.8;    // 规模缩放指数（<1 为次线性）
constexpr int kBudgetMinSeconds = 60;      // 最小预算
constexpr int kBudgetMaxSeconds = 3600;    // 最大预算（1 小时）
```

### 2.3 计算公式

```cpp
int64_t budget_seconds = kBudgetBaseTime;
if (fine.n > kBudgetBaseNodes) {
    double ratio = (double)fine.n / kBudgetBaseNodes;
    budget_seconds = (int64_t)(kBudgetBaseTime * pow(ratio, kBudgetExponent));
}
budget_seconds = clamp(budget_seconds, kBudgetMinSeconds, kBudgetMaxSeconds);
```

**缩放指数 0.8** 的原因：粗化过程每层将问题规模减少 50-70%，有效问题规模的增长是次线性的。

### 2.4 典型值

| 数据集 | 节点数 | 自适应预算 |
|--------|--------|-----------|
| ibm01 | 12k | 90s（基准） |
| 50k | 50k | ~287s |
| case1 | 300k | ~1243s（21 分钟） |
| case8 | 750k | ~2716s（45 分钟） |

## 3. 性能优化：扁平数组

### 3.1 问题

原 k 路实现使用 `vector<vector<int>>` 作为线网计数数组，导致 **749k 次独立堆分配**（每个线网一个 `vector<int>`），内存碎片化严重，缓存未命中率极高。

### 3.2 修复

统一替换为扁平 `vector<int>`，单次连续分配：

```
改造前：vector<vector<int>> count         改造后：vector<int> count
         │                                    │
         ├─ count[0] → [p0, p1, ...]         ├─ [0]: count[0*k+0]
         ├─ count[1] → [p0, p1, ...]         ├─ [1]: count[0*k+1]
         ├─ count[2] → [p0, p1, ...]         ├─ [2]: count[1*k+0]
         ...   (离散堆分配，缓存不友好)        ...   (连续内存，缓存友好)
```

### 3.3 涉及函数

| 函数 | 签名变更 |
|------|---------|
| `build_net_side_counts_kway` | `vector<vector<int>>&` → `vector<int>&` |
| `cut_from_counts_kway` | `vector<vector<int>>` → `vector<int>`, 新增 `num_nets` |
| `delta_cut_for_move_kway` | `vector<vector<int>>` → `vector<int>` |
| `apply_move_kway` | `vector<vector<int>>&` → `vector<int>&`, 新增 `int k` |
| `fm_refine_kway` | 内部 `count` 变量 `vector<vector<int>>` → `vector<int>` |
| `evaluate_cut_kway` | 同步变更 |

## 4. 多路划分（k-way）改造

### 4.1 核心数据结构变更

| 数据项 | 二分版本 | k 路版本 |
|--------|---------|----------|
| 分区向量 | `vector<char> part` (0/1) | `vector<int> part` (0..k-1) |
| 线网计数 | `count_x[net]`, `count_y[net]` | `count[net_idx*k + p]`（扁平数组） |
| 权重追踪 | `int wx, wy` | `vector<int> weight(k)` |
| 平衡边界 | `int min_x, max_x` | `vector<int> min_w[k], max_w[k]` |
| 移动记录 | `char from_side` | `int from_part, int to_part` |
| 增益条目 | `int gain, int node, int version` | 新增 `int to_part` |

### 4.2 重写的函数

| 函数 | 说明 |
|------|------|
| `compute_balance_bounds_kway` | 计算 k 路平衡边界 min_w[k], max_w[k] |
| `build_net_side_counts_kway` | 统计每条线网在各 partition 中的节点数 |
| `cut_from_counts_kway` | 计算 k 路 cut（涉及 ≥2 个 partition 的线网数） |
| `delta_cut_for_move_kway` | 计算节点 u 移动到 to_part 的 cut 变化量 |
| `apply_move_kway` | 执行移动，更新 count 数组和 weight 向量 |

### 4.3 cut 定义

```cpp
// 一条线网被切开 ⇔ 涉及至少 2 个不同 partition
bool is_net_cut(const vector<int> &count_row, int k) {
    int parts = 0;
    for (int p = 0; p < k; ++p)
        if (count_row[p] > 0) parts++;
    return parts >= 2;
}
```

### 4.4 平衡约束

```cpp
void compute_balance_bounds_kway(int total_weight, int k,
                                  vector<int> &min_w, vector<int> &max_w) {
    // 每路平衡范围: [total/k - r*total, total/k + r*total]
    double per_part = (double)total_weight / k;
    double margin = kBalanceRatio * total_weight;
    int min_val = max(0, (int)ceil(per_part - margin));
    int max_val = min(total_weight, (int)floor(per_part + margin));
    min_w.assign(k, min_val);
    max_w.assign(k, max_val);
}
```

`r = kBalanceRatio = 0.02`，当 `r × k = 1` 时严格均分。

### 4.5 三种初始化方法

| 方法 | 说明 | 函数 |
|------|------|------|
| 度数均衡 | 按节点度降序，分配到最轻 partition | `init_degree_balanced_kway` |
| 随机均衡 | 随机顺序，分配到最轻 partition | `init_random_balanced_kway` |
| 小网权重 | 小线网 score 排序，兼顾 score 均衡 | `init_smallnet_balanced_kway` |

核心逻辑统一为：遍历排序后的节点，将其分配到**当前总权重最轻**且**不超过上限**的 partition。

### 4.6 FM 细化（k 路版本）

关键变化：

1. **初始增益计算**：每个节点评估全部 k-1 个目标 partition，选增益最大的入堆
2. **移动选择**：从堆中弹出候选 `(u → to_part)`，检查平衡约束
3. **增量更新**：移动后重新计算受影响邻居到所有目标的增益，选最大入堆
4. **回滚**：无改善时反向移动（`apply_move_kway` 的 `from_part` 和 `to_part` 互换）

```cpp
// 每个节点计算 k-1 个目标的增益，选最大入堆
for (int u = 0; u < hg.n; ++u) {
    int best_g = INT_MIN, best_t = -1;
    for (int p = 0; p < k; ++p) {
        if (p == part[u]) continue;
        int g = -delta_cut_for_move_kway(hg, u, p, part, count, k);
        if (g > best_g) { best_g = g; best_t = p; }
    }
    gain[u] = best_g;
    gain_target[u] = best_t;
    max_heap.push({best_g, u, best_t, gain_version[u]});
}
```

### 4.7 主算法集成

`run_one_restart` 重写为 `run_one_restart_kway`，新增 `int k` 参数。

`my_partition_algorithm`：
- 新增 `int k` 参数（默认 2）
- 新增 `vector<int> *node_partition` 输出参数
- 返回 `int`（最佳 cutsize）
- k=2 时填充 X/Y 集合（向后兼容）
- k>2 时填充 `node_partition` 映射

### 4.8 与二分的兼容性

k=2 时完全兼容原有二分行为：
- `part[u]` 取值 0/1
- X/Y 集合正常填充
- `evaluate()` 仍可用于验证

## 6. 其他修正

### 6.1 Graph::get_or_create_node O(n²) → O(log n)

**问题**：原实现线性扫描 `nodes` 向量查找节点，导致 `O(n²)` 复杂度。

```cpp
// 改造前（O(n) 每次，总 O(n²)）
Node *Graph::get_or_create_node(int index) {
    for(auto node : nodes) {
        if(node->get_index() == index) return node;
    }
    // ...
}

// 改造后（O(log n) 每次，总 O(n log n)）
Node *Graph::get_or_create_node(int index) {
    auto it = node_map.find(index);
    if (it != node_map.end()) return it->second;
    // ...
}
```

**影响**：case1（300k 节点）读取时间从数十分钟降至秒级。

### 6.2 read_benchmark 格式更新

从旧的 `.hgr` 格式（第一行 `net_num node_num`）改为只支持 ICCAD 2021 TopoPart 格式：

```
Line 1: <num_nodes>
Line 2: <num_nets>
Line 3..3+num_nets-1: net descriptions (0-based node IDs)
Remaining: fixed node info (ignored)
```

**0→1 索引转换**：读取时 `node_id++` 将文件的 0 索引转为内部 1 索引。

### 6.3 路径分隔符支持

`main.cpp` 中同时识别 `/` 和 `\` 路径分隔符：
```cpp
size_t slash_pos = base_name.find_last_of("/\\");
```

### 6.4 进度输出

添加运行时进度提示：

## 7. Makefile 变更

### 7.1 编译标志

`-pthread` → `-fopenmp`，启用 OpenMP 支持。

### 7.2 平台兼容的 clean

自动检测 `rm` 是否可用：

```makefile
RMFOUND := $(shell rm --version 2>NUL)
ifdef RMFOUND
RM = rm -f                    # MSYS2/MinGW bash
TARGET = main
else
RM = del /f /q                # Windows cmd
TARGET = main.exe
endif
```

### 7.3 完整 Makefile

```makefile
SRC = main.cpp Graph.cpp Net.cpp Node.cpp solution.cpp evaluate.cpp
OBJ = $(SRC:.cpp=.o)
CXX = g++
CFLAGS = -Wall -Wextra -Werror -std=c++17 -pedantic -fopenmp
CFLAGS += -g

RMFOUND := $(shell rm --version 2>NUL)
ifdef RMFOUND
RM = rm -f
TARGET = main
else
RM = del /f /q
TARGET = main.exe
endif

all: $(TARGET)

$(TARGET): $(OBJ)
	$(CXX) $(CFLAGS) -o $@ $(OBJ)

%.o: %.cpp
	$(CXX) $(CFLAGS) -c $< -o $@

clean:
	$(RM) $(TARGET) $(OBJ)
```
