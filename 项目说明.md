# VLSI Lab1 项目说明

本文档面向刚接触该项目的同学，解释当前目录下 C++ 项目的用途、目录结构、核心代码实现和运行时需要注意的问题。

## 1. 项目整体目标

本项目是 VLSI/FPGA 相关课程的 Lab1，主题是**超图二划分**。

简单理解：

- 输入是一张超图，每个顶点可以理解为电路模块或单元。
- 每条超边 `Net` 可以连接多个顶点，表示这些模块之间有连线关系。
- 程序需要把所有顶点分成两个集合 `X` 和 `Y`。
- 两个集合大小要尽量均衡，本实验要求比例在 `50% ± 2%` 之间。
- 目标是让被切开的超边数量尽量少。

所谓“被切开的超边”，指一条 `Net` 同时连接了 `X` 中的点和 `Y` 中的点。这样的 net 就会产生一次 cut。

## 3. 输入数据格式

benchmark 文件后缀是 `.hgr`。以 `ibm01.hgr` 开头几行为例：

```text
14111 12752
12704 8118
3045 12604
3768 12751
4993 12737
```

第一行：

```text
超边数量 顶点数量
```

例如 `14111 12752` 表示：

- 有 `14111` 条超边；
- 有 `12752` 个顶点。

从第二行开始，每一行表示一条超边连接了哪些顶点。例如：

```text
12704 8118
```

表示当前这条 net 连接顶点 `12704` 和顶点 `8118`。如果一行有更多数字，就表示这条超边连接更多顶点。

## 4. 程序运行主流程

入口在 `lab1/main.cpp`。

核心流程如下：

```cpp
Solution solution;
Graph graph;

solution.read_benchmark(graph, benchmark_name);

set<int> X;
set<int> Y;
solution.my_partition_algorithm(graph, X, Y);

cout << evaluate(graph, "./ibm01_partition.txt") << endl;
```

执行逻辑：

1. 检查命令行参数。程序需要一个 benchmark 文件路径。
2. 创建 `Graph graph`，用于保存超图。
3. 调用 `read_benchmark` 读取 `.hgr` 文件。
4. 打印 node 和 net 数量。
5. 创建两个集合 `X` 和 `Y`。
6. 调用 `my_partition_algorithm` 计算二划分结果。
7. 调用 `evaluate` 读取分区结果文件并计算 cutsize。

需要注意：当前 `main.cpp` 调用了划分算法，但没有把 `X/Y` 写入 `ibm01_partition.txt`。也就是说，当前代码中“算出划分结果”和“评估文件结果”之间还缺少一个输出文件步骤。

## 5. 核心数据结构

### 5.1 Node 顶点类

位置：`lab1/Node.h`、`lab1/Node.cpp`

`Node` 表示一个顶点。

核心成员：

```cpp
int index;
vector<Net *> nets;
```

含义：

- `index`：顶点编号，例如 `1`、`2`、`12752`。
- `nets`：这个顶点连接到哪些超边。

核心函数：

```cpp
Node(int index);
int get_index();
void add_net(Net *net);
vector<Net *> &get_nets();
```

当读取 benchmark 时，如果某条 net 连接了某个 node，就会调用：

```cpp
node->add_net(net);
```

这样每个点都能知道自己属于哪些 net。

### 5.2 Net 超边类

位置：`lab1/Net.h`、`lab1/Net.cpp`

`Net` 表示一条超边。

核心成员：

```cpp
int index;
vector<Node *> nodes;
```

含义：

- `index`：超边编号，从 `0` 开始。
- `nodes`：这条超边连接了哪些顶点。

核心函数：

```cpp
Net(int index);
void add_node(Node *node);
int get_index();
vector<Node *> &get_nodes();
```

读取 benchmark 时，如果某条 net 包含某个 node，就会调用：

```cpp
net->add_node(node);
```

这样每条超边都能知道自己连接了哪些点。

### 5.3 Graph 超图类

位置：`lab1/Graph.h`、`lab1/Graph.cpp`

`Graph` 是整个超图的容器。

核心成员：

```cpp
vector<Node *> nodes;
vector<Net *> nets;
map<int, Node*> node_map;
map<int, Net*> net_map;
```

含义：

- `nodes`：保存所有顶点指针。
- `nets`：保存所有超边指针。
- `node_map`：通过顶点编号快速找到对应 `Node*`。
- `net_map`：通过超边编号快速找到对应 `Net*`。

主要函数：

```cpp
Node *get_or_create_node(int index);
Net *add_net(int index);
```

`get_or_create_node` 的逻辑是：

1. 先检查当前图里有没有这个编号的 node。
2. 如果有，直接返回已有 node。
3. 如果没有，就 `new Node(index)` 创建一个新 node，加入 `nodes` 和 `node_map`。

`add_net` 的逻辑更简单：

1. 创建一个新的 `Net`。
2. 放进 `nets`。
3. 记录到 `net_map`。

## 6. benchmark 读取逻辑

位置：`lab1/solution.cpp` 中的 `Solution::read_benchmark`。

核心代码逻辑：

```cpp
ifstream file(benchmark_name);
iss >> edge_num;
iss >> node_num;

for (int i = 0; i < edge_num; i++) {
    Net *net = graph.add_net(i);

    while (iss >> node_id) {
        Node *node = graph.get_or_create_node(node_id);
        node->add_net(net);
        net->add_node(node);
    }
}
```

逐步解释：

1. 打开输入文件。
2. 读取第一行，得到超边数 `edge_num` 和顶点数 `node_num`。
3. 后面每一行对应一条 net。
4. 每读到一个顶点编号：
   - 从 `Graph` 里拿到或创建对应 `Node`；
   - 把当前 `Net` 加入这个 `Node` 的 net 列表；
   - 把这个 `Node` 加入当前 `Net` 的 node 列表。

这一步结束后，图中会形成双向关联：

```text
Graph
├── Node 1 -> Net A, Net B, ...
├── Node 2 -> Net A, ...
└── Net A  -> Node 1, Node 2, ...
```

## 7. 当前划分算法实现

位置：`lab1/solution.cpp` 中的 `Solution::my_partition_algorithm`。

当前默认启用的是：

```cpp
// Algorithm 2: Greedy single-move improvement
```

也就是“随机均衡初始划分 + 贪心单点移动优化”。

### 7.1 初始化：随机均衡划分

相关函数：

```cpp
init_random_balanced(graph, X, Y, rng);
```

逻辑：

1.  收集所有 node 编号。
2. 用固定随机种子 `42` 打乱顺序。
3. 按照约一半一半的比例，把前半部分放到 `X`，后半部分放到 `Y`。
4. 同时满足 `50% ± 2%` 的平衡约束。

因为随机种子固定为 `42`，所以同一个输入文件每次跑出的初始划分是可复现的。

### 7.2 平衡约束

相关常量：

```cpp
constexpr double kBalanceRatio = 0.02;
```

相关函数：

```cpp
compute_balance_bounds(n, min_x, max_x);
```

它会计算 `X` 集合允许的最小和最大大小：

```text
min_x = ceil((0.5 - 0.02) * n)
max_x = floor((0.5 + 0.02) * n)
```

也就是：

```text
48% * n <= |X| <= 52% * n
```

因为总点数固定，只要 `X` 满足这个范围，`Y` 也会满足。

### 7.3 统计每条 net 在 X/Y 中的点数

相关函数：

```cpp
build_net_counts(graph, X, count_x, count_y);
```

它会为每条 net 统计两个数字：

- `count_x[i]`：第 `i` 条 net 中有多少点在 `X`。
- `count_y[i]`：第 `i` 条 net 中有多少点在 `Y`。

有了这个统计后，判断一条 net 是否被 cut 就很快：

```text
count_x[i] > 0 且 count_y[i] > 0
```

### 7.4 计算移动一个点带来的 cut 变化

相关函数：

```cpp
delta_cut_for_move(node, from_x, count_x, count_y);
```

它计算“把一个点从当前集合移动到另一个集合”以后，cutsize 会变化多少。

返回值含义：

- `delta < 0`：移动后 cut 变少，是好操作。
- `delta = 0`：移动后 cut 不变。
- `delta > 0`：移动后 cut 变多，是坏操作。

对每个与该 node 相连的 net，它会比较移动前后：

```cpp
bool before_cut = (cx > 0 && cy > 0);
bool after_cut = (cx > 0 && cy > 0);
delta += after_cut - before_cut;
```

举例：

- 如果某条 net 原来跨越 `X/Y`，移动后不跨了，则 `delta -= 1`。
- 如果某条 net 原来不跨，移动后跨了，则 `delta += 1`。
- 如果移动前后都跨或都不跨，则 `delta` 不变。

### 7.5 贪心迭代优化

核心循环：

```cpp
for (int iter = 0; iter < kMaxGreedyIters; ++iter) {
    ...
}
```

当前最大迭代次数：

```cpp
constexpr int kMaxGreedyIters = 5;
```

每轮迭代会做：

1. 遍历所有 node。
2. 判断移动这个 node 是否仍满足平衡约束。
3. 计算移动它带来的 `delta`。
4. 找到 `delta` 最小的 node。
5. 如果最好的 `delta < 0`，就真的移动这个 node。
6. 如果没有任何移动可以减少 cut，就提前停止。

这是一种比较直观的局部搜索算法：每一步只做当前看起来最划算的一次移动。

## 8. 评估模块

位置：`lab1/evaluate.cpp`。

### 8.1 calculate_cut

函数：

```cpp
int calculate_cut(Graph graph, set<int> X, set<int> Y);
```

作用：

1. 检查 `X/Y` 是否满足 `50% ± 2%` 的平衡要求。
2. 遍历所有 net。
3. 如果某条 net 同时连接 `X` 和 `Y`，cut 加一。
4. 返回 cutsize。

### 8.2 evaluate

函数：

```cpp
int evaluate(Graph graph, string partition_name);
```

作用：

1. 打开划分结果文件。
2. 逐行读取分区编号。
3. 第 `i` 行对应编号为 `i` 的 node。
4. 如果这一行是 `0`，把 node `i` 放入 `X`。
5. 如果这一行是 `1`，把 node `i` 放入 `Y`。
6. 调用 `calculate_cut` 计算 cutsize。

输出文件格式应类似：

```text
0
1
1
0
...
```

## 9. 当前实现中需要特别注意的问题

### 9.1 分区结果没有写入文件

当前 `my_partition_algorithm` 只填充了内存中的：

```cpp
set<int> X;
set<int> Y;
```

但是 `main.cpp` 后面调用的是：

```cpp
evaluate(graph, "./ibm01_partition.txt")
```

这要求当前目录下已经存在 `ibm01_partition.txt`。如果文件不存在，评估结果就不可靠。

也就是说，当前项目距离完整运行还差一步：

```text
把 X/Y 按 node 编号顺序写入 benchmark_name_partition.txt
```

### 9.2 输出文件名被写死

当前代码无论输入什么 benchmark，都评估：

```text
./ibm01_partition.txt
```

如果运行的是 `ibm02.hgr`，理论上应该输出并评估：

```text
ibm02_partition.txt
```

### 9.3 Graph 被按值传递

当前一些函数签名是：

```cpp
void my_partition_algorithm(Graph graph, set<int> &X, set<int> &Y);
int calculate_cut(Graph graph, set<int> X, set<int> Y);
int evaluate(Graph graph, string partition_name);
```

这里的 `Graph graph` 是按值传递，会复制 `Graph` 对象。由于 `Graph` 内部保存的是 `Node*` 和 `Net*` 指针，复制本身暂时能工作，但对大 benchmark 来说会有额外开销，也容易让初学者误解对象所有权。

更自然的写法通常是：

```cpp
const Graph &graph
```

### 9.4 Graph 没有释放 new 出来的对象

`Graph::get_or_create_node` 和 `Graph::add_net` 使用了 `new`：

```cpp
Node *node = new Node(index);
Net *net = new Net(index);
```

但 `Graph` 析构函数里没有 `delete` 它们。程序短时间运行问题不大，但从 C++ 习惯上看，这是内存泄漏。

### 9.5 get_or_create_node 查找效率可以优化

当前 `get_or_create_node` 会先遍历整个 `nodes`：

```cpp
for (auto node : nodes) {
    if (node->get_index() == index) return node;
}
```

但类里已经有 `node_map`，可以直接用 map 查找，速度会更好。

## 11. 一句话理解整个项目

这个项目的本质是：

```text
读取超图数据 -> 建立 Node/Net 双向连接 -> 先随机平衡划分 -> 贪心移动顶点降低 cut -> 用分区文件计算 cutsize
```

当前代码已经实现了“读取超图”和“内存中的贪心划分”，但“把划分结果写入文件并正确评估对应 benchmark”这一步还没有完整接上。
