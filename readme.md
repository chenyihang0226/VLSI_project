## 1. 实验目标

本实验目标是在给定超图网表上完成二分划分，输出两个划分 $V_1$ 与 $V_2$，在满足平衡约束的前提下最小化割边数量（cutsize）。

平衡约束为：

$$
\frac{1}{2}-\epsilon \le \frac{|V_i|}{|V|} \le \frac{1}{2}+\epsilon, \quad \epsilon=2\%
$$

## 2. 算法逻辑与实现思路

### 2.1 总体框架

本次实现采用“多层次超图划分 + FM 细化”的流程：

1. 将输入网表构建为超图结构（节点、超边、节点关联超边）。
2. 在粗化阶段按匹配规则合并节点，形成多层粗图（最多 5 层），直到规模足够小或粗化收益变弱。
3. 在最粗层进行初始化与局部优化（FM）。
4. 逐层反粗化：将上一层分区映射到下一层，并在每层继续 FM 细化。
5. 在最细层得到最终划分，输出分区文件并调用评估函数计算 cutsize。

### 2.2 关键策略

#### 2.2.1 粗化匹配评分（质量增强）

为提高粗化质量，匹配评分不再只看“共享超边次数”，而是加入：

1. 小网加权：网规模越小，贡献越大（强化强耦合关系）。
2. 度数惩罚：高连接度节点适当降权，避免粗化阶段形成过大超级节点。
3. 同分时优先匹配度更低的候选，提升后续细化可操作性。

#### 2.2.2 多初始化竞争（质量增强）

每次重启在最粗层执行三种初始化并竞争：

1. 度数平衡初始化。
2. 随机平衡初始化。
3. 小网权重平衡初始化。

三种初始化都进入 FM 流程，保留 cut 更小的结果。

#### 2.2.3 FM 细化（速度与质量折中）

细化阶段采用增量式 gain 管理：

1. 用最大堆维护候选移动的 gain。
2. 每次移动后，仅更新受影响节点的 gain（局部增量更新）。
3. 保留 FM 的前缀最优回滚机制，避免负收益累计。

该方式相比全量扫描可降低单步开销，并减少高质量候选被采样遗漏的问题。

#### 2.2.4 时间预算与稳健性

为避免在大规模数据上运行时间失控，算法设置总时间预算（90 秒），在预算内尽可能执行更多层级细化与重启，并始终保存当前最优解。

## 5. 结果分析

### 5.1 有效性

1. 在中小规模数据集（如 IBM03~IBM08、IBM12、IBM13）上，结果明显优于早期基线方法，说明多层次 + 增量 FM 的方向正确。
2. IBM01 结果为 261，相比最优 200 仍有差距，但已显著优于随机/弱局部优化策略。

### 5.2 局限性

1. 在大规模实例（IBM14~IBM18）上，与最优差距明显增大。
2. 主要原因是：
   - 当前仍是启发式近似，容易停在局部最优。
   - 时间预算限制下，大图重启次数与细化深度不足。
   - 粗化质量仍有提升空间（匹配质量决定后续上限）。

## 6. 实验总结与改进方向

### 6.1 本方法优点

1. 架构清晰：多层次流程完整，便于继续迭代。
2. 可扩展：粗化、初始化、细化模块解耦，方便替换策略。
3. 稳定性较好：有平衡约束检查、前缀回滚、时间预算保护。

### 6.2 下一步改进

1. 实现更标准的 FM bucket 结构（双向桶 + 更高效的可行性过滤），进一步提升速度和质量。
2. 引入更深层次粗化与更精细反粗化策略（例如层间自适应 pass 数）。
3. 对大规模图单独设计参数策略（按规模动态调整重启数、候选规模、预算分配）。
4. 在粗化阶段加入更强的结构感知匹配（例如基于超边分布熵或局部割敏感度）。
